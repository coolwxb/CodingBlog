  1. 为什么要加 autoreleasepool{} 或 try{} @finally{} {} .
    答: 这样做是为了在代码中使用这两个宏时能够加上 @ 符号, 实现 @weakify(self) 和 @strongify(self) 这样的效果，将宏替换为实际的实现时, 代码就变成了 @autoreleasepool{} 和 @try , 这两个其实实际上并没有什么实际的作用, 仅仅是为了可以加 “@” 符号而已.
    至于为什么在Debug环境要使用 @autoreleasepool{} 而在Release环境使用 @try{} @finally{} {} 这个是为了在Debug环境下保持编译器的分析报错能力, 而使用 @try{} @finally{} {} 会抑制编译器的这个能力，原本的报错会变成一个警告。
    详细解释请看:
    请谨慎使用@weakify和@strongify
    ReactiveCocoa中weakify与strongify的作用
    Reactive Cocoa中@weakify和@strongify实现分析

  2. typeof(self) 在block代码块内这样写为什么不会造成循环引用(retain cycle)?
    答: typeof 、 __typeof 、 typeof 这三种写法功能是一样的，这三个函数并不是标准C中规定的，它是一种编译器特性。
    个人认为在block代码块中使用 typeof(self) , 是用来获取self这个指针的类型的，而并未使用到self对象，因此不会对self对象造成强引用。
    详细解释请看:
    Stack Overflow

  3. 为什么要在block代码块内写 @strongify(self) ?
    答: 在block代码块内写 @strongify(self) 与block代码块外的 @weakify(self) 一起搭配使用的目的是为了在执行代码块内代码时(注意仅仅是在执行block代码块的代码时)保证self对象不会释放。如果self对象在block代码块的代码执行之前提前释放, 则不适用于此。此解释也同时可以解释问题4。

  4. 以下代码，在执行block之前将view置为nil，但是block代码块内已经对view强引用了，为什么block代码块内的打印还是null?
    Objective-C
    @interface ViewController ()
    @property (copy , nonatomic) void(myblock)();
    @end

@implementation ViewController

  • (void)viewDidLoad {
    [super viewDidLoad];

    UIView *view = [[UIView alloc] init];
    // @weakify(view)
    __weak __typeof(view) weakView = view;
    NSLog(@"view(block块前)>%@", weakView);
    self.myblock = {
    // @strongify(view)
    __strong __typeof(view) view = weakView;
    NSLog(@"view(block块内)    >%@", view);
    };
    view = nil;
    NSLog(@"view(block块后)>%@", weakView);
    self.myblock();
    }
    打印结果:
    Bash
    2017-05-16 14:12:44.512 test[34550:12206532] view(block块前)    >>
    2017-05-16 14:12:44.513 test[34550:12206532] view(block块后)>(null)
    2017-05-16 14:12:44.513 test[34550:12206532] view(block块内)    >(null)
    答: 虽然myblock代码块内对view强引用了, 但是在执行 view=nil 时, myblock的这段实现代码仅仅是存储在内存区，并未执行，所以此时 view=nil 时，view对象的引用计数变为0会立即释放, 当执行到 self.myblock(); 时，block才会对view强引用，而此时view已经释放了，因此打印结果为 null 。
    正确的测试代码如下:
    Objective-C
    @interface ViewController ()
    @property (copy , nonatomic) void(myblock)();
    @end

@implementation ViewController

  • (void)viewDidLoad {
    [super viewDidLoad];

    __block UIView *view = [[UIView alloc] init];
    // @weakify(view)
    __weak __typeof(view) weakView = view;
    NSLog(@"view(block块前)>%@", weakView);
    self.myblock = {
    // @strongify(view)
    __strong __typeof(view) strongView = weakView;
    view = nil;
    NSLog(@"view(block块内)    >%@", strongView);
    };
    NSLog(@"view(block块后)>%@", weakView);
    self.myblock();
    NSLog(@"view(block执行后)    >%@", weakView);
    }
    打印结果:
    Bash
    2017-05-16 15:17:57.729 test[34648:12246862] view(block块前)>>
    2017-05-16 15:17:57.730 test[34648:12246862] view(block块后)    >>
    2017-05-16 15:17:57.730 test[34648:12246862] view(block块内)>>
    2017-05-16 15:17:57.730 test[34648:12246862] view(block执行后)    >(null)
